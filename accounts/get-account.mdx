### Path
<ParamField path="account_id" type="string" required>
  The unique identifier of the account to retrieve.
</ParamField>

### Query
<ParamField query="include" type="array">
  Comma-separated list of related entities to include in the response. Valid values: `balance`, `identity`, `ledger`.

  ```
  /accounts/{account_id}?include=balance,identity,ledger
  ```
</ParamField>

### Response
<ResponseField name="account_id" type="string">
  Unique identifier for the account.
</ResponseField>
<ResponseField name="name" type="string">
  The account name.
</ResponseField>
<ResponseField name="number" type="string">
  The account number.
</ResponseField>
<ResponseField name="bank_name" type="string">
  The name of the bank.
</ResponseField>
<ResponseField name="currency" type="string">
  The account currency.
</ResponseField>
<ResponseField name="balance_id" type="string">
  The ID of the linked balance.
</ResponseField>
<ResponseField name="identity_id" type="string">
  The ID of the linked identity.
</ResponseField>
<ResponseField name="ledger_id" type="string">
  The ID of the linked ledger.
</ResponseField>
<ResponseField name="created_at" type="timestamp">
  ISO 8601 formatted timestamp of when the account was created.
</ResponseField>
<ResponseField name="meta_data" type="object">
  Custom metadata associated with the account.
</ResponseField>

<RequestExample>
```bash
curl --request GET \
  --url http://localhost:5001/accounts/acc_123456789 \
  --header 'X-blnk-key: <api-key>'
```
