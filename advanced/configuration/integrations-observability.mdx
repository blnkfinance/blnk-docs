## Overview

Use these settings to configure outgoing notifications, telemetry, and observability features.

---

## Notification configuration

The notification configuration controls Slack alerts and outgoing webhooks.

<CodeGroup>
```json blnk.json
{
  "notification": {
    "slack": {
      "webhook_url": ""
    },
    "webhook": {
      "url": "",
      "headers": {}
    }
  }
}
```

```bash .env
BLNK_SLACK_WEBHOOK_URL=
BLNK_WEBHOOK_URL=
BLNK_WEBHOOK_HEADERS={}
```
</CodeGroup>

| Attribute | Type | Description | Environment variable |
| :-- | :-- | :-- | :-- |
| `slack.webhook_url` | string | Slack webhook URL used for notifications. | `BLNK_SLACK_WEBHOOK_URL` |
| `webhook.url` | string | Application endpoint that receives webhook notifications. | `BLNK_WEBHOOK_URL` |
| `webhook.headers` | object | Custom HTTP headers added to webhook requests. | `BLNK_WEBHOOK_HEADERS` |

---

## Telemetry and observability

Use telemetry and observability settings to enable tracing, metrics, and related runtime visibility.

<CodeGroup>
```json blnk.json
{
  "enable_telemetry": true,
  "enable_observability": true
}
```

```bash .env
BLNK_ENABLE_TELEMETRY=true
BLNK_ENABLE_OBSERVABILITY=true
```
</CodeGroup>

| Attribute | Type | Description | Environment variable |
| :-- | :-- | :-- | :-- |
| `enable_telemetry` | boolean | Sends trace data for monitoring. Defaults to `false`. | `BLNK_ENABLE_TELEMETRY` |
| `enable_observability` | boolean | Enables additional observability features such as metrics and detailed logging. Defaults to `false`. | `BLNK_ENABLE_OBSERVABILITY` |

<Info>
  Telemetry and observability are separate settings. You can enable one without enabling
  the other.
</Info>
