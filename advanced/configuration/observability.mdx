This page covers the top-level flags that turn on trace export and broader observability behavior in Blnk. These settings are independent from [queue monitoring](/advanced/monitoring-port), which exposes a separate HTTP port for queue inspection.

---

## Telemetry and observability flags

Use these boolean flags to opt in to distributed tracing and additional runtime visibility.

<CodeGroup>
```bash blnk.env
BLNK_ENABLE_TELEMETRY=false
BLNK_ENABLE_OBSERVABILITY=false
```

```json blnk.json
{
  "enable_telemetry": false,
  "enable_observability": false
}
```
</CodeGroup>

| | Description | Default |
| :-- | :-- | :-- |
| `BLNK_ENABLE_TELEMETRY` | When `true`, Blnk emits trace data suitable for monitoring and APM tools. | `false` |
| `BLNK_ENABLE_OBSERVABILITY` | When `true`, Blnk enables additional observability features such as metrics and more detailed logging. | `false` |

<Note>
**Please note:** Telemetry and observability are separate settings. You can enable one without enabling the other, depending on whether you need traces, metrics and deeper logs, or both.
</Note>
