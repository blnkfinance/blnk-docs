This page covers the top-level flags that control **operational** observability (OpenTelemetry and metrics), optional **remote monitoring export**, and **product** heartbeat telemetry. These settings are independent from [queue monitoring](/advanced/monitoring-port), which exposes a separate HTTP port for queue inspection.

---

## Telemetry and observability flags

Use these boolean flags to opt in to traces and metrics, or to product usage telemetry.

<CodeGroup>
```bash blnk.env
BLNK_ENABLE_TELEMETRY=false
BLNK_ENABLE_OBSERVABILITY=false
BLNK_MONITORING_DSN=https://your-write-key@monitoring.example.com/your-project-id
```

```json blnk.json
{
  "enable_telemetry": false,
  "enable_observability": false,
  "monitoring_dsn": "https://your-write-key@monitoring.example.com/your-project-id"
}
```
</CodeGroup>

| | Description | Default |
| :-- | :-- | :-- |
| `BLNK_ENABLE_TELEMETRY` | When `true`, Blnk sends **product heartbeat / usage** telemetry (for example to PostHog) so the team can understand usage trends. It is **not** the same switch as distributed traces or `/metrics`. | `false` |
| `BLNK_ENABLE_OBSERVABILITY` | When `true`, Blnk initializes the **OpenTelemetry** SDK, exports **traces**, and serves the **HTTP `/metrics` endpoint** (Prometheus-style metrics) as wired in the server. Use this for APM, dashboards, and SRE visibility. | `false` |
| `BLNK_MONITORING_DSN` | HTTPS DSN for remote export of logs, traces, and metrics. | none |

### `BLNK_MONITORING_DSN`

**`BLNK_MONITORING_DSN`** works only when **`BLNK_ENABLE_OBSERVABILITY`** is `true`. If observability is off, the DSN is ignored.

When a valid DSN is configured:

1. **Logs** — Application logs are forwarded asynchronously. Sensitive values are redacted from log messages and fields (including bearer tokens, DSNs, URLs, and query parameters).
2. **Traces** — OpenTelemetry traces are exported over OTLP/HTTP to your monitoring backend.
3. **Metrics** — OpenTelemetry metrics are exported over OTLP/HTTP on a 60-second interval, in addition to the local Prometheus `/metrics` endpoint.

<Tip>
Use `BLNK_MONITORING_DSN` when you want a managed monitoring sink without running your own OpenTelemetry Collector. For self-hosted OTLP export, you can still use standard `OTEL_EXPORTER_OTLP_*` environment variables alongside observability.
</Tip>
