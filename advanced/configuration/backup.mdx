## Overview

Use these settings to configure local backup paths and S3-compatible backup storage.

---

## S3 backup configuration

These settings control where backups are stored and how Blnk authenticates to S3 or an S3-compatible service.

<CodeGroup>
```json blnk.json
{
  "backup_dir": "",
  "aws_access_key_id": "",
  "aws_secret_access_key": "",
  "s3_endpoint": "",
  "s3_bucket_name": "",
  "s3_region": ""
}
```

```bash .env
BLNK_BACKUP_DIR=
BLNK_AWS_ACCESS_KEY_ID=
BLNK_AWS_SECRET_ACCESS_KEY=
BLNK_S3_ENDPOINT=
BLNK_S3_BUCKET_NAME=
BLNK_S3_REGION=
```
</CodeGroup>

| Attribute | Type | Description | Environment variable |
| :-- | :-- | :-- | :-- |
| `backup_dir` | string | Directory path used for backup storage. | `BLNK_BACKUP_DIR` |
| `aws_access_key_id` | string | AWS access key ID for authentication. | `BLNK_AWS_ACCESS_KEY_ID` |
| `aws_secret_access_key` | string | AWS secret access key for authentication. | `BLNK_AWS_SECRET_ACCESS_KEY` |
| `s3_endpoint` | string | Custom endpoint for S3-compatible services. | `BLNK_S3_ENDPOINT` |
| `s3_bucket_name` | string | Name of the S3 bucket used for backups. | `BLNK_S3_BUCKET_NAME` |
| `s3_region` | string | AWS region for the S3 bucket. | `BLNK_S3_REGION` |

<Tabs>
  <Tab title="Best practices">
    1. Separate backup storage from your main application runtime.
    2. Use retention rules so old backups do not accumulate without limits.
    3. Verify restore flows regularly, not just backup creation.
    4. Use least-privilege IAM credentials for S3 access.
  </Tab>
</Tabs>
