## Overview

Use this endpoint to revoke unused or expired API keys to maintain good security hygiene. 

Revoking an API key immediately revokes access, so ensure you update your applications with a new API key before revoking the old one.

<Info>
Available on version 0.10.1 or later. Requires master key authentication (the `server.secret_key` from your configuration).
</Info>

<Key />

### Path Parameters
<ParamField path="id" type="string" required>
  The unique identifier of the API key to revoke.
</ParamField>

### Response
Returns 204 No Content upon successful revocation.

<RequestExample>
```bash
curl -X DELETE \
  -H "X-Blnk-Key: your_master_key" \
  https://YOUR_BLNK_INSTANCE_URL:5001/api-keys/key_abc123def456
```
</RequestExample>
