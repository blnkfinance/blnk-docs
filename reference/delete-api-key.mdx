Use this endpoint to revoke unused or expired API keys to maintain good security hygiene. 

Revoking an API key immediately revokes access, so ensure you update your applications with a new API key before revoking the old one.

<Info>
Available on version 0.10.1 or later. Requires the master key or an API key with `api-keys:delete` scope.

Starting in **0.14.3**, non-master callers can revoke keys only within their own owner context. Passing a different `owner` returns `403 Forbidden`.
</Info>

<Key />

### Path Parameters
<ParamField path="id" type="string" required>
  The unique identifier of the API key to revoke.
</ParamField>

### Query Parameters
<ParamField query="owner" type="string">
  Unique identifier of the owner. **Required** when you authenticate with the master key. When you authenticate with a non-master API key, omit this parameter to revoke within the caller's owner context, or pass a matching owner value.
</ParamField>

### Response
Returns 204 No Content upon successful revocation.

<RequestExample>
```json
curl --request DELETE \
  --url 'http://localhost:5001/api-keys/<api-key-id>?owner=owner_id' \
  --header 'X-blnk-key: <api-key>'
```
</RequestExample>
<ResponseExample>
```json 204
{}
```
</ResponseExample>
