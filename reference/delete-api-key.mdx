Use this endpoint to revoke unused or expired API keys to maintain good security hygiene. 

Revoking an API key immediately revokes access, so ensure you update your applications with a new API key before revoking the old one.

<Info>
Available on version 0.10.1 or later. Requires the master key or an API key with `api-keys:delete` scope.
</Info>

<Key />

### Path Parameters
<ParamField path="id" type="string" required>
  The unique identifier of the API key to revoke.
</ParamField>

### Query Parameters
<ParamField query="owner" type="string">
  Unique identifier of the owner whose key you want to revoke.
</ParamField>

### Response
Returns 204 No Content upon successful revocation.

<RequestExample>
```json
curl -X DELETE "http://localhost:5001/api-keys/<api-key-id>?owner=owner_id" \
  -H "X-blnk-key: <api-key>"
```
</RequestExample>
<ResponseExample>
```json 204
{}
```
</ResponseExample>
