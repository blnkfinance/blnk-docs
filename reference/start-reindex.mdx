Start a reindex to populate Typesense with data from your existing database. Use when you spin up a new Typesense instance alongside a database that already contains records.

<Key />

### Response
<ResponseField name="status" type="string">
  Current state: `in_progress` or `completed`.
</ResponseField>
<ResponseField name="phase" type="string">
  Current phase (e.g. `indexing_transactions`) or `done` when complete.
</ResponseField>
<ResponseField name="total_records" type="integer">
  Total number of records to index.
</ResponseField>
<ResponseField name="processed_records" type="integer">
  Number of records processed so far.
</ResponseField>
<ResponseField name="started_at" type="string">
  ISO 8601 timestamp when reindex began.
</ResponseField>
<ResponseField name="completed_at" type="string">
  ISO 8601 timestamp when reindex finished. Only present when `status` is `completed`.
</ResponseField>
<ResponseField name="errors" type="array">
  Array of error messages. Only present when an error occurs during reindex.
</ResponseField>

<RequestExample>
```bash
curl -X POST "http://YOUR_BLNK_INSTANCE_URL/search/reindex" \
  -H "X-Blnk-Key: YOUR_API_KEY"
```
</RequestExample>

<ResponseExample>
```json 200
{
    "status": "in_progress",
    "phase": "indexing_transactions",
    "total_records": 2145709,
    "processed_records": 2145709,
    "started_at": "2026-02-12T12:13:10.448612096Z"
}
```
</ResponseExample>
