Use `client.Reconciliation.Run` to start batch reconciliation.

<Steps>
<Step title="Call the method">

```go client.Reconciliation.Run wrap
reconResp, resp, err := client.Reconciliation.Run(blnkgo.RunReconData{
	UploadID: "upload_8c700d1b-09c0-4ef4-9ab1-a0decf3d0aa3",
	Strategy: blnkgo.ReconciliationStrategyOneToOne,
	DryRun: true,
	GroupingCriteria: blnkgo.CriteriaFieldReference,
	MatchingRuleIDs: []string{
		"rule_890bdbc4-467f-4670-8424-5667e41daf29",
	},
})
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `UploadID` | `string` | Upload ID returned by `client.Reconciliation.`<br />`Upload`. |
| `Strategy` | `blnkgo.`<br />`ReconciliationStrategy` | Match strategy: `ReconciliationStrategy`<br />`OneToOne`, `ReconciliationStrategy`<br />`OneToMany`, or `ReconciliationStrategy`<br />`ManyToOne`. |
| `DryRun` | `bool` | Set `true` to test without writing results to the ledger. |
| `GroupingCriteria` | `blnkgo.`<br />`CriteriaField` | Field used to group records, for example `CriteriaField`<br />`Reference`. |
| `MatchingRuleIDs` | `[]string` | IDs of the matching rules to apply. |

</Step>

<Step title="Response">

```json 200 OK wrap
{
  "reconciliation_id": "rec_a8f2c91d-1234-4abc-9def-0123456789ab"
}
```

#### Go response (`*blnkgo.StartReconciliationResponse`)

| Field | Type | Description |
| :-- | :-- | :-- |
| `ReconciliationID` | `string` | ID of the reconciliation run. |

</Step>
</Steps>
