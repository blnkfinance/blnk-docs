Use `blnk.transactions.get` to fetch a single [transaction](/transactions/introduction) by its transaction ID.

<Note>
The Python SDK does not expose a list method for transactions. Use [Search](/sdks/python/search/search-db) or get by reference instead.
</Note>

<Steps>
<Step title="Call the method">

```python blnk.transactions.get wrap
response = blnk.transactions.get(
  "txn_93c3f6dc-9c2c-46b0-89c5-1814cef4d0e4",
)
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `transactionId` | `str` | Transaction ID to retrieve. |

</Step>

<Step title="Use the transaction details">

Read `response.data` for status, amounts, source, destination, and metadata. Use the transaction ID for refunds, inflight updates, or lineage queries.

</Step>

<Step title="Response">

```json 200 OK wrap expandable
{
  "precise_amount": 300,
  "amount": 3,
  "rate": 0,
  "precision": 100,

  "destination": "bln_e92a1ba7-92c7-427c-8c8f-bfdca55c875d",
  "reference": "ref_04551509-d7d3-4eab-a1fd-2eb12809b5a4",
  "currency": "BTC",

  "status": "APPLIED",
  "hash": "0b9c25fb5b00d6c71cb4ca87026bf6dc316e63353d3330deb588bd0b3d74dcc0",

    "payment_method": "card"

```

| Field | Type | Description |
| :-- | :-- | :-- |
| `transaction_id` | `str` | Unique ID for the transaction. |
| `status` | `str` | Current [status](/transactions/transaction-lifecycle#statuses) of the transaction. |
| `source` | `str` | Source balance. |
| `destination` | `str` | Destination balance. |
| `reference` | `str` | Reference you supplied at create time. |
| `meta_data` | `dict` | Metadata stored with the transaction. |

</Step>
