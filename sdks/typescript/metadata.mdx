## Overview

Use `blnk.Metadata` to attach or merge `meta_data` on any Blnk resource. Pass the resource ID (ledger, balance, transaction, or identity) as the first argument.

For concepts, see [Metadata overview](/metadata/overview).

| Method | API Reference | Description |
| :--- | :--- | :--- |
| `update` | [Update metadata](/reference/update-metadata) | `POST /{id}/metadata` with `{ meta_data }` |

<Tip>
Every method returns an `ApiResponse<T>`. Check `response.status` and `response.data` after each call. See [Error handling](/sdks/typescript#error-handling) on the install page.
</Tip>

---

## Update metadata

`blnk.Metadata.update` merges metadata keys onto an existing resource.

```typescript wrap
const response = await blnk.Metadata.update(
  'ldg_073f7ffe-9dfd-42ce-aa50-d1dca1788adc',
  {
    meta_data: {
      project_owner: 'Acme LLC',
      update_status: 'Approved',
    },
  },
);

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
} else {
  console.log(response.data.meta_data);
}
```

<Icon icon="sliders-horizontal" size={16} color="#808080" className="cli-section-icon" /> **Parameters**

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | string | Yes | Resource ID (`ldg_…`, `bln_…`, `txn_…`, or `idt_…`). |
| `data.meta_data` | object | Yes | Key-value metadata to merge. |

See [Update metadata](/reference/update-metadata) for request fields, response shape, and validation.
