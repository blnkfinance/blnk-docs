## Overview

Use `blnk.System` for operational endpoints that are not tied to a ledger resource. Start with `health` to confirm Core is reachable before posting transactions.

| Method | API Reference | Description |
| :--- | :--- | :--- |
| `health` | [Health check](/reference/get-health) | `GET /health` — instance status |

<Tip>
Every method returns an `ApiResponse<T>`. Check `response.status` and `response.data` after each call. See [Error handling](/sdks/typescript#error-handling) on the install page.
</Tip>

---

## Health check

`blnk.System.health` probes Core availability. Does not require authentication on local instances.

```typescript wrap
const response = await blnk.System.health();

if (response.status !== 200 || response.data?.status !== 'UP') {
  console.error('Core is not reachable:', response.message);
  process.exit(1);
}

console.log('Core status:', response.data.status);
```

Use this in startup scripts, deployment probes, and before running integration tests.

See [Health check](/reference/get-health) for response fields and deployment notes.
