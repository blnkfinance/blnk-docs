Use `blnk.Transactions.createBulk` to create many [transactions](/transactions/bulk-transactions) in one request.

<Steps>
<Step title="Call the method">

```typescript blnk.Transactions.createBulk wrap
const response = await blnk.Transactions.createBulk({
  atomic: true,
  transactions: [
    {
      amount: 1000,
      precision: 100,
      currency: 'USD',
      source: '@source_1',
      destination: '@dest_1',
      reference: 'bulk_001',
      description: 'Payment 1',
    },
    {
      amount: 2000,
      precision: 100,
      currency: 'USD',
      source: '@source_2',
      destination: '@dest_2',
      reference: 'bulk_002',
      description: 'Payment 2',
    },
  ],
});
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `transactions` | `object[]` | Transaction objects to create; references must be unique within the batch. |
| `atomic` | `boolean` | When `true`, every transaction succeeds or fails together. |
| `inflight` | `boolean` | When `true`, transactions are created in `INFLIGHT` status for a later commit or void. |
| `run_async` | `boolean` | When `true`, Core processes the batch in the background and returns `status: 'queued'`. |
| `skip_queue` | `boolean` | When `true`, transactions bypass the queue and process synchronously. |

</Step>

<Step title="Save the batch ID">

Use `response.data.batch_id` as `parent_transaction` to find child transactions, or with updateStatus to commit or void an inflight batch.

</Step>

<Step title="Response">

<CodeGroup>
```json Sync wrap
{
  "batch_id": "bulk_c62f200b-905f-4983-a349-cadd279234aa",
  "status": "applied",
  "transaction_count": 4
}
```

```json Async (run_async: true) wrap
{
  "batch_id": "bulk_c62f200b-905f-4983-a349-cadd279234aa",
  "status": "queued",
  "message": "Bulk transaction processing started"
}
```
</CodeGroup>

| Field | Type | Description |
| :-- | :-- | :-- |
| `batch_id` | `string` | Identifier for the batch. Use as parent_transaction to find child transactions, or with updateStatus to commit or void an inflight batch. |
| `status` | `string` | `applied`, `inflight`, or `queued` depending on the batch. |
| `transaction_count` | `number` | Number of transactions when processed synchronously. |
| `message` | `string` | Status message when run_async is true. |

</Step>
</Steps>
