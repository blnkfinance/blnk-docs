Use `blnk.Transactions.updateStatus` to commit or void an [inflight](/transactions/inflight) transaction or batch.

<Steps>
<Step title="Call the method">

<Tabs>
<Tab title="Commit">

```typescript blnk.Transactions.updateStatus wrap
const response = await blnk.Transactions.updateStatus(
  'txn_c4e70eb8-e4d6-4e04-a2e2-92a43b969e0c',
  {
    status: 'commit',
  },
);
```

</Tab>

<Tab title="Partial commit">

```typescript blnk.Transactions.updateStatus wrap
const response = await blnk.Transactions.updateStatus(
  'txn_c4e70eb8-e4d6-4e04-a2e2-92a43b969e0c',
  {
    status: 'commit',
    amount: 500
  },
);
```

</Tab>

<Tab title="Void">

```typescript blnk.Transactions.updateStatus wrap
const response = await blnk.Transactions.updateStatus(
  'txn_c4e70eb8-e4d6-4e04-a2e2-92a43b969e0c',
  {
    status: 'void',
  },
);
```

</Tab>
</Tabs>

| Field | Type | Description |
| :-- | :-- | :-- |
| `transactionOrBatchId` | `string` | Inflight transaction ID, or batch ID from createBulk. |
| `status` | `string` | `commit` to apply funds, or `void` to cancel and roll back. |
| `amount` | `number` | Partial commit amount. See [Partial commits](/transactions/inflight#partial-commits). |
| `precise_amount` | `number` | Precision-applied partial commit amount. Takes precedence over amount. |

</Step>

<Step title="Confirm the outcome">

Check `response.data.status` for `COMMIT` or `VOID`. Use `parent_transaction` to link the result back to the original inflight transaction.

</Step>

| Field | Type | Description |
| :-- | :-- | :-- |
| `transaction_id` | `string` | ID of the new transaction created by the commit or void. |
| `parent_transaction` | `string` | Original inflight transaction this resolves. |
| `status` | `string` | `COMMIT` or `VOID`. |
| `precise_amount` | `number` | Amount applied (amount × precision). |

</Step>
