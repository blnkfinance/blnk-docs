Use `blnk.Transactions.create` to post a [transaction](/transactions/introduction) between a source and destination balance.

<Steps>
<Step title="Call the method">

<Tabs>
<Tab title="Standard">

```typescript blnk.Transactions.create wrap
const response = await blnk.Transactions.create({
  amount: 1250.34,
  precision: 100,
  currency: 'USD',
  source: '@WorldUSD',
  destination: '@MyBalance',
  reference: 'ref_2ye281ewiu',
  description: 'Customer payment',
  allow_overdraft: false,
  skip_queue: false,
});

```

</Tab>

<Tab title="Inflight">

To hold funds until you commit or void, include:

```typescript wrap
inflight: true,
inflight_expiry_date: '2025-03-01T00:00:00Z',
inflight_commit_date: '2025-02-28T12:00:00Z',
```

See [Update inflight](/sdks/typescript/transactions/update-inflight).

</Tab>

<Tab title="Multiple sources">

To debit more than one source, include:

```typescript wrap
sources: [
  {
    identifier: '@CardWallet',
    distribution: '40%',
  },
  {
    identifier: '@MainWallet',
    distribution: 'left',
  },
],
destination: '@MerchantBalance',
atomic: true,
```

See [Multiple sources](/transactions/multiple-sources).

</Tab>

<Tab title="Multiple destinations">

To credit more than one destination, include:

```typescript wrap
source: '@MainWallet',
destinations: [
  {
    identifier: '@SavingsWallet',
    distribution: '60%',
  },
  {
    identifier: '@SpendingWallet',
    distribution: 'left',
  },
],
atomic: true,
```

See [Multiple destinations](/transactions/multiple-destinations).

</Tab>

<Tab title="Scheduled">

To schedule, include:

```typescript wrap
scheduled_for: '2025-03-01T09:00:00Z',
```

See [Scheduling](/transactions/scheduling).

</Tab>

<Tab title="Backdated">

To record a past transaction date, include:

```typescript wrap
effective_date: '2025-01-15T12:00:00Z',
```

See [Backdated transactions](/transactions/backdated-transactions).

</Tab>
</Tabs>

| Field | Type | Description |
| :-- | :-- | :-- |
| `amount` | `number` | Transaction amount. Use `precise_amount` for values with more than 15 digits. |
| `precise_amount` | `number` | Amount with [precision](/transactions/precision) already applied. |
| `currency` | `string` | Currency of the transaction. |
| `precision` | `number` | Precision for the currency (for example `100` for cents). |
| `reference` | `string` | Unique reference for the transaction. |
| `source` | `string` | Balance sending the amount. Prefix `@` for an [internal balance](/balances/internal-balances). |
| `destination` | `string` | Balance receiving the amount. |
| `description` | `string` | Narration of the transaction. |

<Expandable title="Optional parameters">

| Field | Type | Description |
| :-- | :-- | :-- |
| `allow_overdraft` | `boolean` | Allow the source balance to go negative. See [Overdrafts](/transactions/overdrafts). |
| `inflight` | `boolean` | Hold the transaction until you commit or void it. See [Inflight](/transactions/inflight). |
| `inflight_expiry_date` | `string` | When an inflight transaction expires and rolls back. |
| `inflight_commit_date` | `string` | When Core auto-commits the inflight transaction. |
| `scheduled_for` | `string` | Date and time to post a [scheduled](/transactions/scheduling) transaction. |
| `effective_date` | `string` | Actual date the transaction occurred. See [Backdated transactions](/transactions/backdated-transactions). |
| `sources` | `object[]` | Source legs for a multiple sources transaction. |
| `destinations` | `object[]` | Destination legs for a multiple destinations transaction. |
| `sources[].identifier` | `string` | Balance ID or internal balance name for a source leg. |
| `destinations[].identifier` | `string` | Balance ID or internal balance name for a destination leg. |
| `distribution` | `string` | Percentage, fixed amount, or `left` for a split leg. |
| `precise_distribution` | `string` | Fixed split-leg amount in minor units. |
| `skip_queue` | `boolean` | Bypass the transaction queue and process directly. |
| `atomic` | `boolean` | For split legs, apply all legs together or none. |
| `meta_data` | `Record<string, unknown>` | Optional metadata. Tag funds with `BLNK_LINEAGE_PROVIDER` for [fund lineage](/transactions/lineage). |

</Expandable>

</Step>

<Step title="Save the transaction ID">

Save the transaction ID from the response for later retrieval, refunds, or inflight updates.

</Step>

| Field | Type | Description |
| :-- | :-- | :-- |
| `transaction_id` | `string` | Unique ID for the transaction. |
| `status` | `string` | Transaction [status](/transactions/transaction-lifecycle#statuses) (for example `QUEUED`, `APPLIED`). |
| `precise_amount` | `number` | `amount` × `precision`. |
| `source` | `string` | Resolved source balance. |
| `destination` | `string` | Resolved destination balance. |
| `hash` | `string` | [Hash](/transactions/hash) of the transaction details. |
| `created_at` | `string` | Date and time the transaction was created. |

</Step>
</Steps>
