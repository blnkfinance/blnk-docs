Read the original value of one tokenized field with `blnk.Identity.detokenizeField`. The stored identity record is not modified. Pass the field name in **PascalCase** in the path (for example `EmailAddress`). See [Tokenize identity](/sdks/typescript/methods/identities/tokenize-identity) for why tokenization uses PascalCase, not snake_case.

For tokenization concepts, see [PII tokenization](/identities/pii-tokenization) and the [Detokenize field](/reference/detokenize-field) API reference.

---

## Method

```typescript
blnk.Identity.detokenizeField(id, field)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Identity` |
| **SDK method** | `detokenizeField` |
| **HTTP** | `GET /identities/{identity_id}/detokenize/{field}` |
| **Returns** | `Promise<ApiResponse<DetokenizeIdentityFieldResp \| null>>` |

---

## When to use

Call `detokenizeField` when authorized workflows need the plaintext value of one protected field, for example support verification or regulated data access.

---

## Example

```typescript wrap
const response = await blnk.Identity.detokenizeField(
  'idt_8cc22560-5ace-4989-8953-fcbad7947d48',
  'EmailAddress',
);

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.field, response.data.value);
```

---

## Parameters

| Argument | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Yes | Identity ID. |
| `field` | `string` | Yes | PascalCase field name to detokenize. |

---

## Response

Returns the field name and original value in `response.data` when `status` is `200`. See [Detokenize field](/reference/detokenize-field) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that the identity ID and field name are present. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `id`
- Missing or empty `field`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
