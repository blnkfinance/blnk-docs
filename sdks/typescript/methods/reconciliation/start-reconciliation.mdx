Start a batch reconciliation run with `blnk.Reconciliation.run`. Pass the `upload_id` from `upload`, a strategy, grouping field, and matching rule IDs.

For workflow and strategies, see the [Reconciliation overview](/reconciliations/overview) guide and the [Start reconciliation](/reference/start-reconciliation) API reference.

---

## Method

```typescript
blnk.Reconciliation.run(data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Reconciliation` |
| **SDK method** | `run` |
| **HTTP** | `POST /reconciliation/start` |
| **Returns** | `Promise<ApiResponse<RunReconResp \| null>>` |

---

## When to use

Call `run` after you have uploaded external data and created matching rules. Use `dry_run: true` to test matching without writing results to the ledger.

---

## Example

```typescript wrap
const response = await blnk.Reconciliation.run({
  upload_id: 'upload_8c700d1b-09c0-4ef4-9ab1-a0decf3d0aa3',
  strategy: 'one_to_one',
  dry_run: true,
  grouping_criteria: 'reference',
  matching_rule_ids: ['rule_abc123'],
});

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data);
```

<Tip>
CLI equivalent: `blnk reconciliations`. See [CLI reconciliations](/cli/reconciliations).
</Tip>

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `upload_id` | `string` | Yes | Upload ID from `blnk.Reconciliation.upload`. |
| `strategy` | `string` | Yes | `one_to_one`, `one_to_many`, or `many_to_one`. |
| `dry_run` | `boolean` | Yes | `true` to test without persisting matches; `false` to apply results. |
| `grouping_criteria` | `string` | Yes | Field used to group records (for example `amount`, `reference`). |
| `matching_rule_ids` | `string[]` | Yes | One or more rule IDs from `createMatchingRule`. |

For strategy behavior, see [Reconciliation strategies](/reconciliations/strategies).

---

## Response

Returns reconciliation run details in `response.data` when `status` is `201`. See [Start reconciliation](/reference/start-reconciliation) and [Get reconciliation](/sdks/typescript/methods/reconciliation/get-reconciliations) for status and count fields.

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 201) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
