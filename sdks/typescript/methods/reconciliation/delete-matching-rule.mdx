Delete a matching rule with `blnk.Reconciliation.deleteMatchingRule`. Remove rules you no longer use in reconciliation runs.

For when to create or update rules instead, see the [Matching rules](/reconciliations/matching-rules) guide and the [Delete matching rule](/reference/delete-matching-rule) API reference.

---

## Method

```typescript
blnk.Reconciliation.deleteMatchingRule(ruleId)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Reconciliation` |
| **SDK method** | `deleteMatchingRule` |
| **HTTP** | `DELETE /reconciliation/matching-rules/{rule_id}` |
| **Returns** | `Promise<ApiResponse<DeleteMatchingRuleResp \| null>>` |

---

## When to use

Call `deleteMatchingRule` when a rule is obsolete or was created in error. Confirm no scheduled jobs still reference the `rule_id` before deleting.

---

## Example

```typescript wrap
const response = await blnk.Reconciliation.deleteMatchingRule('rule_abc123');

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.message);
```

---

## Parameters

| Argument | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `ruleId` | `string` | Yes | Matching rule ID to delete. |

---

## Response

Returns a confirmation message in `response.data` when `status` is `200`. See [Delete matching rule](/reference/delete-matching-rule) for the response shape.

---

## Client-side validation

Before sending a request to Core, the SDK checks that `ruleId` is present. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `ruleId`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
