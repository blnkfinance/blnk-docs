Fetch reconciliation run details with `blnk.Reconciliation.get`. Read status, matched and unmatched counts, and timing after batch or instant reconciliation.

For workflow context, see the [Reconciliation overview](/reconciliations/overview) guide and the [View reconciliation details](/reference/get-reconciliations) API reference.

---

## Method

```typescript
blnk.Reconciliation.get(reconciliationId)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Reconciliation` |
| **SDK method** | `get` |
| **HTTP** | `GET /reconciliation/{reconciliation_id}` |
| **Returns** | `Promise<ApiResponse<ReconciliationResp \| null>>` |

---

## When to use

Call `get` after `run` or `runInstant` when you need the current status, match counts, or completion time for a reconciliation job.

---

## Example

```typescript wrap
const response = await blnk.Reconciliation.get(
  'rec_a8f2c91d-1234-4abc-9def-0123456789ab',
);

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(
  response.data.status,
  response.data.matched_transactions,
  response.data.unmatched_transactions,
);
```

<Tip>
CLI equivalent: `blnk reconciliations list`. See [CLI reconciliations](/cli/reconciliations).
</Tip>

---

## Parameters

| Argument | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `reconciliationId` | `string` | Yes | Reconciliation ID from a `run` or `runInstant` response. |

---

## Response

Returns a reconciliation object in `response.data` when `status` is `200`. Key fields include `reconciliation_id`, `status`, `matched_transactions`, `unmatched_transactions`, `is_dry_run`, `started_at`, and `completed_at`. See [View reconciliation details](/reference/get-reconciliations) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that `reconciliationId` is present. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `reconciliationId`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
