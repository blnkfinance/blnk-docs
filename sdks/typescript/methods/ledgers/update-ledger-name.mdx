Rename an existing ledger with `blnk.Ledgers.update`. Only the `name` field is writable on this endpoint. Metadata and other ledger properties are not updated here.

For field definitions and response shape, see the [General ledger](/ledgers/general-ledger) guide and the [Update ledger name](/reference/update-ledger-name) API reference.

---

## Method

```typescript
blnk.Ledgers.update(id, data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Ledgers` |
| **SDK method** | `update` |
| **HTTP** | `PUT /ledgers/{id}` |
| **Returns** | `Promise<ApiResponse<CreateLedgerResp \| null>>` |

---

## When to use

Call `update` when a ledger display name should change, for example after a product rebrand or when correcting a name entered during setup. The `ledger_id` does not change.

---

## Example

```typescript wrap
const response = await blnk.Ledgers.update(
  'ldg_049495c6-356e-4ebc-a45e-60d1e1e16afb',
  { name: 'Updated Customer Savings Ledger' },
);

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.ledger_id, response.data.name);
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Yes | Ledger ID to update. |
| `name` | `string` | Yes | New ledger name. |

<Tip>
Request field names use **snake_case** to match the HTTP API. The request body only accepts `name`.
</Tip>

---

## Response

Returns the updated ledger object in `response.data` when `status` is `200`. Key fields include `ledger_id`, `name`, `created_at`, and `meta_data`. See [Update ledger name](/reference/update-ledger-name) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that required arguments are present and correctly typed. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `id`
- Missing or empty `name`

```typescript wrap
// Client-side validation failure (no request sent to Core)
const response = await blnk.Ledgers.update('', { name: 'New name' });

if (response.status === 400) {
  console.error('Validation failed:', response.message);
}
```

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
