Commit multiple independently-created inflight transactions in one request with `blnk.Transactions.bulkCommitInflight`. Each request accepts up to 100 items.

For per-item failure codes and when to use batch `updateStatus` instead, see the [Inflight](/transactions/inflight#bulk-inflight-updates) guide and the [Bulk commit inflight](/reference/bulk-commit-inflight) API reference.

---

## Method

```typescript
blnk.Transactions.bulkCommitInflight(data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Transactions` |
| **SDK method** | `bulkCommitInflight` |
| **HTTP** | `POST /transactions/inflight/bulk/commit` |
| **Returns** | `Promise<ApiResponse<BulkInflightCommitResp \| null>>` |

---

## When to use

Call `bulkCommitInflight` when you have multiple independently-created inflight transactions (not from a single bulk batch) that need committing together. Omit `amount` and `precise_amount` on an item to commit the full remaining inflight amount. When both are set, `precise_amount` takes precedence.

<Info>Available in Core 0.14.2 and later.</Info>

---

## Example

```typescript wrap expandable
const response = await blnk.Transactions.bulkCommitInflight({
  transactions: [
    { transaction_id: 'txn_11111111-1111-4111-8111-111111111111' },
    {
      transaction_id: 'txn_22222222-2222-4222-8222-222222222222',
      amount: 40,
    },
    {
      transaction_id: 'txn_33333333-3333-4333-8333-333333333333',
      precise_amount: 125034,
    },
  ],
});

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.succeeded, response.data.failed);
for (const result of response.data.results) {
  console.log(result.transaction_id, result.status, result.code);
}
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `transactions` | `array` | Yes | Up to 100 inflight transactions to commit. Each item requires `transaction_id`. |
| `transaction_id` | `string` | Yes | Inflight transaction ID to commit (per item). |
| `amount` | `float` | No | Partial commit amount. When omitted, Blnk commits the full remaining inflight amount. |
| `precise_amount` | `integer` | No | Precision-applied partial commit amount. Takes precedence over `amount` when both are set. |

<Tip>
Request field names use **snake_case** to match the HTTP API (`transaction_id`, `precise_amount`, etc.).
</Tip>

---

## Response

Returns a bulk commit result in `response.data` when `status` is `200`. Key fields include `succeeded`, `failed`, and a `results` array with per-transaction outcomes. See [Bulk commit inflight](/reference/bulk-commit-inflight) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that required arguments are present and correctly typed. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `transactions` array
- More than 100 items in `transactions`
- Missing `transaction_id` on any item

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```


For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
