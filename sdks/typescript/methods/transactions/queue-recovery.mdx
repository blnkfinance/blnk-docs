Manually trigger recovery of stuck queued transactions with `blnk.Transactions.recoverQueue`. Optionally pass a `threshold` duration (for example `5m`, `1h`) to control which transactions are re-enqueued.

For queue behavior and concurrency, see the [Transactions](/transactions/introduction) guide and the [Queue recovery](/reference/queue-recovery) API reference.

---

## Method

```typescript
blnk.Transactions.recoverQueue(options?)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Transactions` |
| **SDK method** | `recoverQueue` |
| **HTTP** | `POST /transactions/recover` |
| **Returns** | `Promise<ApiResponse<QueueRecoveryResp \| null>>` |

---

## When to use

Call `recoverQueue` when transactions appear stuck in the queue and you need to manually re-enqueue them for processing. Useful for operational recovery after outages or when monitoring detects delayed processing.

---

## Example

```typescript wrap
// Log Core error details when the request fails
const response = await blnk.Transactions.recoverQueue({ threshold: '5m' });

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.recovered, response.data.threshold);
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `threshold` | `string` | No | Age threshold for stuck transactions (for example `5m`, `1h`). Default: `2m`. Minimum: 2 minutes. |

---

## Response

Returns a recovery result in `response.data` when `status` is `200`. Key fields include `recovered` (number of transactions re-enqueued) and `threshold`. See [Queue recovery](/reference/queue-recovery) for all response fields.

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
const response = await blnk.Transactions.recoverQueue();

if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```


For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
