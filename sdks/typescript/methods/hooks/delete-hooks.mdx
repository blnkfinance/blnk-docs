Delete a hook with `blnk.Hooks.delete`. Remove webhooks you no longer use in your transaction flow.

For hook lifecycle, see the [Create hooks](/hooks/create-hooks) guide and the [Delete hooks](/reference/delete-hooks) API reference.

<Warning>
Hook management requires the **master key** in `X-Blnk-Key`. Regular API keys return `403 Forbidden`.
</Warning>

---

## Method

```typescript
blnk.Hooks.delete(id)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Hooks` |
| **SDK method** | `delete` |
| **HTTP** | `DELETE /hooks/{id}` |
| **Returns** | `Promise<ApiResponse<DeleteHookResp \| null>>` |

---

## When to use

Call `delete` when a webhook endpoint is retired or was created in error. Disable a hook with `update` first if you may need the configuration later.

---

## Example

```typescript wrap
const response = await blnk.Hooks.delete('hk_test_123');

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.message);
```

---

## Parameters

| Argument | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Yes | Hook ID to delete. |

---

## Response

Returns a confirmation message in `response.data` when `status` is `200`. See [Delete hooks](/reference/delete-hooks) for the response shape.

---

## Client-side validation

Before sending a request to Core, the SDK checks that `id` is present. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `id`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
