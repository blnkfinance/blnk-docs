Fetch a single balance by `balance_id` with `blnk.LedgerBalances.get`. Returns credit and debit totals, currency, and the current net balance.

For balance concepts and response fields, see the [Balances](/balances/introduction) guide and the [Get balance](/reference/get-balance) API reference.

---

## Method

```typescript
blnk.LedgerBalances.get(id, options?)
```

| | |
| :--- | :--- |
| **Service** | `blnk.LedgerBalances` |
| **SDK method** | `get` |
| **HTTP** | `GET /balances/{balance_id}` |
| **Returns** | `Promise<ApiResponse<CreateLedgerBalanceResp \| null>>` |

---

## When to use

Call `get` when you have a `balance_id` from a `create` response, transaction record, or support tooling, and need the current balance amount and metadata. Pass `{ from_source: true }` to reconstruct the balance from transactions instead of snapshots.

---

## Example

```typescript wrap
const response = await blnk.LedgerBalances.get(
  'bln_5ce86029-3c2e-4e2a-aae2-7fb931ca4c4f',
);

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.balance_id, response.data.balance);
```

Reconstruct from transactions instead of snapshots:

```typescript wrap
const fromSource = await blnk.LedgerBalances.get(
  'bln_5ce86029-3c2e-4e2a-aae2-7fb931ca4c4f',
  { from_source: true },
);
```

<Tip>
CLI equivalent: `blnk balances list --id <balance-id>`. See [CLI balances](/cli/balances).
</Tip>

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Yes | Balance ID (for example `bln_5ce86029-3c2e-4e2a-aae2-7fb931ca4c4f`). |
| `from_source` | `boolean` | No | When `true`, reconstruct the balance from transactions instead of snapshots. |

For `from_source` behavior, see [Balance from source](/reference/balance-from-source).

---

## Response

Returns a balance object in `response.data` when `status` is `200`. Key fields include `balance_id`, `balance`, `currency`, `credit_balance`, and `debit_balance`. See [Get balance](/reference/get-balance) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK validates optional request options when you pass them. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Invalid `from_source` type when `options` is provided

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
const response = await blnk.LedgerBalances.get('bln_missing');

if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
