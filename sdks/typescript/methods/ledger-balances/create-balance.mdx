Create a balance on an existing ledger with `blnk.LedgerBalances.create`. Core returns a `balance_id` you use as `source` or `destination` on transactions.

For balance concepts and field definitions, see the [Balances](/balances/introduction) guide and the [Create balance](/reference/create-balance) API reference.

---

## Method

```typescript
blnk.LedgerBalances.create(data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.LedgerBalances` |
| **SDK method** | `create` |
| **HTTP** | `POST /balances` |
| **Returns** | `Promise<ApiResponse<CreateLedgerBalanceResp \| null>>` |

---

## When to use

Call `create` after you have a `ledger_id`. Each balance is a wallet or account on that ledger. Use the returned `balance_id` when posting transactions or linking an identity.

---

## Example

```typescript wrap
const response = await blnk.LedgerBalances.create({
  ledger_id: 'ldg_049495c6-356e-4ebc-a45e-60d1e1e16afb',
  currency: 'USD',
});

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.balance_id);
```

To enable [fund lineage](/transactions/lineage), pass `identity_id`, `track_fund_lineage: true`, and an optional `allocation_strategy` (`FIFO`, `LIFO`, or `PROPORTIONAL`):

```typescript wrap
const lineageResponse = await blnk.LedgerBalances.create({
  ledger_id: 'ldg_049495c6-356e-4ebc-a45e-60d1e1e16afb',
  identity_id: 'idt_3b63c8da-af29-4cc3-ad38-df17d87456e6',
  currency: 'USD',
  track_fund_lineage: true,
  allocation_strategy: 'FIFO',
});
```

<Tip>
CLI equivalent: `blnk balances create`. See [CLI balances](/cli/balances).
</Tip>

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `ledger_id` | `string` | Yes | Ledger the balance belongs to. |
| `currency` | `string` | Yes | Currency or asset class for the balance. |
| `identity_id` | `string` | No | Identity to link to the balance. Required when `track_fund_lineage` is `true`. |
| `track_fund_lineage` | `boolean` | No | Enable fund lineage tracking on this balance. |
| `allocation_strategy` | `string` | No | How tagged provider funds are allocated on spend: `FIFO`, `LIFO`, or `PROPORTIONAL`. |
| `meta_data` | `object` | No | Optional key-value metadata. |

<Tip>
Request field names use **snake_case** to match the HTTP API (`ledger_id`, `track_fund_lineage`, not `ledgerId`).
</Tip>

For optional fields and response fields, see [Create balance](/reference/create-balance).

---

## Response

Returns a balance object in `response.data` when `status` is `201`. Key fields include `balance_id`, `ledger_id`, `currency`, and `balance`. See [Create balance](/reference/create-balance) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that required arguments are present and correctly typed. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or invalid `ledger_id`
- Missing or invalid `currency`
- Invalid `identity_id`, `meta_data`, `track_fund_lineage`, or `allocation_strategy`

```typescript wrap
// Client-side validation failure (no request sent to Core)
const response = await blnk.LedgerBalances.create({
  ledger_id: '',
  currency: 'USD',
});

if (response.status === 400) {
  console.error('Validation failed:', response.message);
}
```

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 201) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
