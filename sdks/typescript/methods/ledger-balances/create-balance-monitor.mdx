Create a balance monitor with `blnk.BalanceMonitor.create`. When the condition is met, Core sends a webhook to your callback URL.

For setup and webhook behavior, see the [Balance monitoring](/balances/balance-monitoring) guide and the [Create balance monitor](/reference/create-balance-monitor) API reference.

---

## Method

```typescript
blnk.BalanceMonitor.create(data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.BalanceMonitor` |
| **SDK method** | `create` |
| **HTTP** | `POST /balance-monitors` |
| **Returns** | `Promise<ApiResponse<MonitorDataResp \| null>>` |

---

## When to use

Call `create` to alert on balance changes, for example low-balance warnings, spending limits, or operational thresholds. Attach the monitor to an existing `balance_id`.

---

## Example

```typescript wrap
const response = await blnk.BalanceMonitor.create({
  balance_id: 'bln_5ce86029-3c2e-4e2a-aae2-7fb931ca4c4f',
  condition: {
    field: 'credit_balance',
    operator: '>',
    value: 1000,
    precision: 100,
  },
  call_back_url: 'https://api.example.com/balance-monitor',
  description: 'Low balance alert',
});

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.monitor_id);
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `balance_id` | `string` | Yes | Balance to monitor. |
| `condition` | `object` | Yes | Threshold rule with `field`, `operator`, `value`, and `precision`. |
| `condition.field` | `string` | Yes | Balance field to watch (for example `credit_balance`, `debit_balance`, `balance`). |
| `condition.operator` | `string` | Yes | Comparison operator: `>`, `<`, or `=`. |
| `condition.value` | `number` | Yes | Threshold amount. |
| `condition.precision` | `number` | Yes | Currency precision (for example `100` for cents). |
| `call_back_url` | `string` | No | Webhook URL when the condition is met. |
| `description` | `string` | No | Human-readable monitor description. |

<Tip>
Request field names use **snake_case** to match the HTTP API (`balance_id`, `call_back_url`, not `balanceId`).
</Tip>

For all condition fields and response shape, see [Create balance monitor](/reference/create-balance-monitor).

---

## Response

Returns a monitor object in `response.data` when `status` is `201`. Key fields include `monitor_id`, `balance_id`, and `condition`. See [Create balance monitor](/reference/create-balance-monitor) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that required arguments are present and correctly typed. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or invalid `balance_id`
- Missing or invalid `condition` (field, operator, value, precision)
- Invalid `description` or `call_back_url` when provided

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 201) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
