Trigger daily balance snapshot processing with `blnk.LedgerBalances.createSnapshot`. Core processes balances in batches and returns a status message when the job starts.

For setup and scheduling concepts, see the [Balance snapshots](/balances/balance-snapshots) guide and the [Balance snapshots](/reference/balances-snapshots) API reference.

---

## Method

```typescript
blnk.LedgerBalances.createSnapshot(options?)
```

| | |
| :--- | :--- |
| **Service** | `blnk.LedgerBalances` |
| **SDK method** | `createSnapshot` |
| **HTTP** | `POST /balances-snapshots` |
| **Returns** | `Promise<ApiResponse<CreateBalanceSnapshotResponse \| null>>` |

---

## When to use

Call `createSnapshot` to run snapshot processing on demand, for example during operational maintenance or when you need snapshots before querying historical balances. Omit `batch_size` or pass `0` to use the server default (1000).

---

## Example

```typescript wrap
const response = await blnk.LedgerBalances.createSnapshot({ batch_size: 500 });

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.message);
```

Use the server default batch size:

```typescript wrap
const defaultBatch = await blnk.LedgerBalances.createSnapshot();
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `batch_size` | `number` | No | Balances processed per batch. Omit or pass `0` for the server default. Must be positive when set. |

---

## Response

Returns a status message in `response.data` when `status` is `200`. See [Balance snapshots](/reference/balances-snapshots) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK validates optional request options when you pass them. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Negative `batch_size`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 200) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
