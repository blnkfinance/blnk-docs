Create a scoped API key with `blnk.ApiKeys.create`. Store the plaintext `key` from the response immediately; Core does not return it again on list.

For scopes and auth model, see [Secure Blnk](/advanced/secure-blnk) and the [Create API key](/reference/create-api-key) API reference.

---

## Method

```typescript
blnk.ApiKeys.create(data)
```

| | |
| :--- | :--- |
| **Service** | `blnk.ApiKeys` |
| **SDK method** | `create` |
| **HTTP** | `POST /api-keys` |
| **Returns** | `Promise<ApiResponse<ApiKeyResp \| null>>` |

---

## When to use

Call `create` when provisioning a service account or integration that should not use the master key. Grant the minimum scopes required for the workload.

---

## Example

```typescript wrap
const response = await blnk.ApiKeys.create({
  name: 'Service Account',
  owner: 'merchant_a',
  scopes: ['ledgers:read', 'balances:write'],
  expires_at: '2026-12-31T23:59:59Z',
});

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.api_key_id, response.data.key);
```

<Warning>
`response.data.key` is shown only at creation. List responses return a hashed value, not the plaintext secret.
</Warning>

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `name` | `string` | Yes | Descriptive key or service account name. |
| `owner` | `string` | Yes | Owner identifier you assign (team, service, or user ID). |
| `scopes` | `string[]` | Yes | Non-empty list of permissions (for example `ledgers:read`). |
| `expires_at` | `string` | Yes | ISO 8601 expiration timestamp (for example `2026-12-31T23:59:59Z`). |

For available scopes, see [Understanding scopes](/advanced/secure-blnk#understanding-scopes).

---

## Response

Returns an API key object in `response.data` when `status` is `201`. Key fields include `api_key_id`, `key`, `name`, `owner_id`, `scopes`, and `expires_at`. See [Create API key](/reference/create-api-key) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK checks that required arguments are present and correctly typed. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Missing or empty `name` or `owner`
- Missing or empty `scopes` array
- Invalid `expires_at` (must be ISO 8601)

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 201) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
