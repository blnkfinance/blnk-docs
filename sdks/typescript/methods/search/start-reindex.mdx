Start a Typesense reindex job with `blnk.Search.startReindex`. Core rebuilds the search index from your database asynchronously. Poll progress with [`getReindexStatus`](/sdks/typescript/methods/search/get-reindex).

For when to reindex, see the [Typesense reindex guide](/search/typesense/reindex) and the [Start reindex](/reference/start-reindex) API reference.

---

## Method

```typescript
blnk.Search.startReindex(options?)
```

| | |
| :--- | :--- |
| **Service** | `blnk.Search` |
| **SDK method** | `startReindex` |
| **HTTP** | `POST /search/reindex` |
| **Returns** | `Promise<ApiResponse<StartReindexResponse \| null>>` |

---

## When to use

Call `startReindex` after spinning up a new Typesense instance or when indexed documents are out of sync with Postgres. Use after bulk data imports or infrastructure changes.

---

## Example

```typescript wrap
const response = await blnk.Search.startReindex({ batch_size: 1000 });

if (response.status !== 202 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
  if (response.data) console.error(response.data);
  return;
}

console.log(response.data.message, response.data.progress.status);
```

Use the Core default batch size:

```typescript wrap
const defaultBatch = await blnk.Search.startReindex();
```

---

## Parameters

| Field | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `batch_size` | `number` | No | Records processed per batch. Omit to send `{}` and use the Core default. Must be a positive integer when set. |

---

## Response

Returns a reindex job response in `response.data` when `status` is `202`. Key fields include `message` and `progress`. See [Start reindex](/reference/start-reindex) for all response fields.

---

## Client-side validation

Before sending a request to Core, the SDK validates optional request options when you pass them. If validation fails, the SDK returns `status: 400` with a message in `response.message`. No HTTP request is made.

Common failures for this method:

- Non-positive `batch_size`

---

## HTTP errors

When Core rejects the request, inspect `response.status`, `response.message`, and `response.data`:

```typescript wrap
// Log Core error details when the request fails
if (response.status !== 202) {
  console.error(response.status, response.message);
  console.error(response.data);
}
```

For timeout, network, and error body formats, see [Error handling](/sdks/typescript#error-handling).
