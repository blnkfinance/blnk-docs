## Overview

Use `blnk.Reconciliation` to match external statements against ledger records. Upload a file, define how rows match transactions, then start a reconciliation run.

For workflow and concepts, see the [Reconciliation overview](/reconciliations/overview).

| Method | API Reference | Description |
| :--- | :--- | :--- |
| `upload` | [Upload data](/reference/upload-data) | Upload external CSV or statement data |
| `createMatchingRule` | [Create matching rule](/reference/create-matching-rule) | Define how rows match ledger transactions |
| `updateMatchingRule` | [Update matching rule](/reference/update-matching-rule) | Change an existing matching rule |
| `deleteMatchingRule` | [Delete matching rule](/reference/delete-matching-rule) | Remove a matching rule |
| `run` | [Start reconciliation](/reference/start-reconciliation) | Run reconciliation on an uploaded file |
| `runInstant` | [Instant reconciliation](/reference/instant-reconciliation) | Reconcile inline external rows without upload |
| `get` | [View reconciliation details](/reference/get-reconciliations) | Fetch reconciliation status by ID |

<Tip>
Every method returns an `ApiResponse<T>`. Check `response.status` and `response.data` after each call. See [Error handling](/sdks/typescript#error-handling) on the install page.
</Tip>

## Upload reconciliation data

`blnk.Reconciliation.upload` uploads a CSV or other external data file for reconciliation. Pass a local file path and a source label.

```typescript wrap
const response = await blnk.Reconciliation.upload(
  'statement.csv',
  'stripe',
);

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
}
```

For workflow and concepts, see the [Reconciliation overview](/reconciliations/overview).

See [Upload data](/reference/upload-data) for request fields, response shape, and validation.

## Create a matching rule

`blnk.Reconciliation.createMatchingRule` defines criteria for matching uploaded rows to ledger transactions.

```typescript wrap
const response = await blnk.Reconciliation.createMatchingRule({
  name: 'Amount match',
  description: 'Match by amount',
  criteria: [{ field: 'amount', operator: 'equals', allowable_drift: 0.01 }],
});

if (response.status !== 201 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
}
```

For criteria and examples, see [Matching rules](/reconciliations/matching-rules).

See [Create matching rule](/reference/create-matching-rule) for request fields, response shape, and validation.

## Update a matching rule

`blnk.Reconciliation.updateMatchingRule` changes an existing rule's name, description, or criteria.

```typescript wrap
const response = await blnk.Reconciliation.updateMatchingRule('rule_abc123', {
  name: 'Amount match (updated)',
  description: 'Match by amount with 1% drift',
  criteria: [{ field: 'amount', operator: 'equals', allowable_drift: 0.01 }],
});

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
}
```

<Icon icon="sliders-horizontal" size={16} color="#808080" className="cli-section-icon" /> **Parameters**

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `ruleId` | string | Yes | The matching rule ID to update. |
| `data` | object | Yes | Full matcher payload (`name`, `criteria`, optional `description`). |

For criteria operators and drift, see [Matching rules](/reconciliations/matching-rules).

See [Update matching rule](/reference/update-matching-rule) for request fields, response shape, and validation.

## Delete a matching rule

`blnk.Reconciliation.deleteMatchingRule` removes a matching rule you no longer need.

```typescript wrap
const response = await blnk.Reconciliation.deleteMatchingRule('rule_abc123');

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
} else {
  console.log(response.data.message);
}
```

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |

## Run reconciliation

`blnk.Reconciliation.run` starts a reconciliation run against a previously uploaded file.

```typescript wrap
const response = await blnk.Reconciliation.run({
  upload_id: 'upl_f3a8b2c1-9d4e-4a7b-8c6e-1f2d3e4a5b6c',
  strategy: 'one_to_one',
  dry_run: true,
  grouping_criteria: 'amount',
  matching_rule_ids: ['rule_abc123'],
});

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
}
```

<Tip>
CLI equivalent: `blnk reconciliations`. See [CLI reconciliations](/cli/reconciliations).
</Tip>

See [Start reconciliation](/reference/start-reconciliation) for request fields, response shape, and validation.

## Run instant reconciliation

`blnk.Reconciliation.runInstant` reconciles inline external transactions without uploading a file first.

```typescript wrap
const response = await blnk.Reconciliation.runInstant({
  external_transactions: [
    {
      id: 'txn_1',
      amount: 5.49,
      reference: 'INV-2023-002',
      currency: 'GBP',
      description: 'Card payment',
      date: '2024-11-15T14:25:30Z',
      source: 'bank-api',
    },
  ],
  strategy: 'one_to_one',
  dry_run: true,
  matching_rule_ids: ['rule_abc123'],
});

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
}
```

For workflow and concepts, see [Instant reconciliation](/reconciliations/overview#option-2-instant-reconciliation).

See [Instant reconciliation](/reference/instant-reconciliation) for request fields, response shape, and validation.

## Get reconciliation details

`blnk.Reconciliation.get` fetches one reconciliation run by ID — status, matched/unmatched counts, and timing.

```typescript wrap
const response = await blnk.Reconciliation.get('rec_a8f2c91d-1234-4abc-9def-0123456789ab');

if (response.status !== 200 || !response.data) {
  console.error(`Error ${response.status}: ${response.message}`);
} else {
  console.log(response.data.status, response.data.matched_transactions);
}
```

<Icon icon="sliders-horizontal" size={16} color="#808080" className="cli-section-icon" /> **Parameters**

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `reconciliationId` | string | Yes | The reconciliation ID from a `run` or `runInstant` response. |

<Tip>
CLI equivalent: `blnk reconciliations list` browses reconciliation jobs in a table. See [CLI reconciliations](/cli/reconciliations).
</Tip>

See [View reconciliation details](/reference/get-reconciliations) for request fields, response shape, and validation.
