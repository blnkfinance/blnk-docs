Use `blnk.reconciliation().runInstant` for this operation.

<Steps>
<Step title="Call the method">

```java wrap
String ruleId = "rule_073f7ffe-9dfd-42ce-aa50-d1dca1788adc";

ExternalTransaction external = ExternalTransaction.create()
    .id("ext_001")
    .amount(42.50)
    .reference("INV-001")
    .currency("USD")
    .description("Instant recon")
    .date("2025-02-15T10:30:00Z")
    .source("integration-test");

ApiResponse<JsonNode> response = blnk.reconciliation().runInstant(
    RunInstantReconData.create()
        .dryRun(true)
        .strategy("one_to_one")
        .matchingRuleIds(List.of(ruleId))
        .externalTransactions(List.of(external)));
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `external_transactions` | `List<`<br />`ExternalTransaction>` | External transaction rows built with `ExternalTransaction.`<br />`create()`. |
| `external_transactions.`<br />`[].id` | `String` | Unique external transaction ID. |
| `external_transactions.`<br />`[].amount` | `Number` | Transaction amount. |
| `external_transactions.`<br />`[].reference` | `String` | Unique transaction reference. |
| `external_transactions.`<br />`[].currency` | `String` | ISO 4217 currency code. |
| `external_transactions.`<br />`[].description` | `String` | Description of the transaction. |
| `external_transactions.`<br />`[].date` | `String` | Transaction date and time. |
| `external_transactions.`<br />`[].source` | `String` | Origin of the transaction data. |
| `strategy` | `String` | Match strategy: `one_to_one`, `one_to_many`, or `many_to_one`. |
| `matching_rule_ids` | `List<String>` | IDs of the matching rules to apply. |

<Expandable title="Optional parameters">

| Field | Type | Description |
| :-- | :-- | :-- |
| `dry_run` | `boolean` | Set `true` to test without writing results to the ledger. |
| `grouping_criteria` | `String` | Field used to group records during matching. |

</Expandable>

</Step>

<Step title="Save the reconciliation ID">

Use `response.data().get("reconciliation_id").asText()` with `blnk.reconciliation().get` to poll status and match counts.

</Step>

<Step title="Response">

```json 200 OK wrap
{
  "reconciliation_id": "rec_a8f2c91d-1234-4abc-9def-0123456789ab"
}
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `reconciliation_id` | `String` | ID of the reconciliation run. Use it with `blnk.`<br />`reconciliation().get` to check status. |

</Step>
</Steps>
