Use `blnk.transactions().bulkVoidInflight` for this operation.

<Steps>
<Step title="Call the method">

<Tabs>
<Tab title="Queued void">

```java wrap
List<String> transactionIds = List.of("txn_f482a1b3-6c2d-4e89-a17b-3d5e8f2a1c94");

ApiResponse<JsonNode> response = blnk.transactions().bulkVoidInflight(
    BulkVoidInflightRequest.create()
        .transactionIds(transactionIds));
```

</Tab>

<Tab title="Synchronous void">

Pass `skipQueue(true)` to process each void synchronously instead of queuing.

```java wrap
List<String> transactionIds = List.of("txn_f482a1b3-6c2d-4e89-a17b-3d5e8f2a1c94");

ApiResponse<JsonNode> response = blnk.transactions().bulkVoidInflight(
    BulkVoidInflightRequest.create()
        .skipQueue(true)
        .transactionIds(transactionIds));
```

</Tab>
</Tabs>

| Field | Type | Description |
| :-- | :-- | :-- |
| `transaction_ids` | `List<String>` | Up to 100 inflight transaction IDs to void. |
| `skip_queue` | `boolean` | When `true`, process synchronously instead of queuing. |

</Step>

<Step title="Review per-item results">

Check `response.data().get("results")` for each transaction's outcome. Failed items include a code and message without stopping the rest of the batch.

</Step>

<Step title="Response">

```json 200 OK wrap expandable
{
  "succeeded": 2,
  "failed": 1,
  "results": [
    {
      "transaction_id": "txn_11111111-1111-4111-8111-111111111111",
      "status": "succeeded"
    },
    {
      "transaction_id": "txn_22222222-2222-4222-8222-222222222222",
      "status": "succeeded"
    },
    {
      "transaction_id": "txn_33333333-3333-4333-8333-333333333333",
      "status": "failed",
      "code": "ALREADY_COMMITTED",
      "message": "cannot void. Transaction already committed"
    }
  ]
}
```

| Field | Type | Description |
| :-- | :-- | :-- |
| `succeeded` | `Number` | Number voided successfully. |
| `failed` | `Number` | Number that failed. |
| `results` | `List<Map<String, Object>>` | Per-transaction outcome in request order. |
| `results[].status` | `String` | `succeeded` or `failed`. |
| `results[].code` | `String` | Failure code (for example `NOT_FOUND`, `ALREADY_COMMITTED`, `NOT_INFLIGHT`). |
| `results[].message` | `String` | Human-readable failure message. |

</Step>
</Steps>
