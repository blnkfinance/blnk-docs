Use `blnk.transactions().updateStatus` for this operation.

<Steps>
<Step title="Call the method">

<Tabs>
<Tab title="Commit">

```java wrap
String transactionId = "txn_f482a1b3-6c2d-4e89-a17b-3d5e8f2a1c94";

ApiResponse<JsonNode> response = blnk.transactions().updateStatus(
    transactionId,
    UpdateTransactionStatus.create().status("commit"));
```

</Tab>

<Tab title="Partial commit">

```java wrap
String transactionId = "txn_f482a1b3-6c2d-4e89-a17b-3d5e8f2a1c94";

ApiResponse<JsonNode> response = blnk.transactions().updateStatus(
    transactionId,
    UpdateTransactionStatus.create()
        .status("commit")
        .amount(500));
```

</Tab>

<Tab title="Void">

```java wrap
String transactionId = "txn_f482a1b3-6c2d-4e89-a17b-3d5e8f2a1c94";

ApiResponse<JsonNode> response = blnk.transactions().updateStatus(
    transactionId,
    UpdateTransactionStatus.create().status("void"));
```

</Tab>
</Tabs>

| Java builder | Wire field | Type | Description |
| :-- | :-- | :-- | :-- |
| (path argument) | `id` | `String` | Inflight transaction ID, or batch ID from `createBulk`. |
| `status()` | `status` | `String` | `commit` to apply funds, or `void` to cancel and roll back. |
| `amount()` | `amount` | `Number` | Partial commit amount. See [Partial commits](/transactions/inflight#partial-commits). |
| `preciseAmount()` | `precise_amount` | `Number` | Precision-applied partial commit amount. Takes precedence over `amount()`. |

</Step>

<Step title="Confirm the outcome">

Check `response.data().get("status").asText()` for `COMMIT` or `VOID`. Use `parent_transaction` to link the result back to the original inflight transaction.

</Step>


| Field | Type | Description |
| :-- | :-- | :-- |
| `transaction_id` | `String` | ID of the new transaction created by the commit or void. |
| `parent_transaction` | `String` | Original inflight transaction this resolves. |
| `status` | `String` | `COMMIT` or `VOID`. |
| `precise_amount` | `Number` | Amount applied (amount × precision). |

</Step>
</Steps>
