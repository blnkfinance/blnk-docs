## Installing Skills

<Steps>
<Step title="Add Skills">
From your project (or any directory where you use the LLM), run:

```bash wrap
npx skills add https://docs.blnkfinance.com
```

The CLI detects tools such as Cursor and Claude Code, and installs the skills for you. Confirm when prompted if you want them on more than one tool.
</Step>

<Step title="Confirm they loaded">
To use the skill, start with `/blnk` and ask any question. That is the root entry point; it routes to the other Blnk skills for architecture, money movement, and implementation.
</Step>

<Step title="Connect the Docs MCP">
To improve the LLM's knowledge base with live Docs lookup, connect the Docs MCP in your LLM tool:

```
https://docs.blnkfinance.com/mcp
```
</Step>
</Steps>

## Try it: build a wallet product

To give it a try, open a chat in your project and describe what you are building in plain language, the same way you would brief a teammate.

Say you are adding customer wallets: people deposit from a bank, hold a USD balance, and can send money to each other.

<Steps>
<Step title="Describe the product">
Paste something like this:

```text wrap
We're adding customer USD wallets.

- Each customer gets a wallet balance
- They deposit from a bank account
- They can transfer money to another customer
- No overdraft

Design the money movement first, then help me implement it.

/blnk
```

You do not need to name ledgers, balances, or API fields yourself.
</Step>

<Step title="Answer a few plain questions">
The LLM should ask short, product-level questions (for example who owns the funds before a deposit clears, or whether transfers are instant). 

Answer as best as you can. Confirm defaults when they sound right.
</Step>

<Step title="Review the money map">
Before any integration code, the LLM should give you a money movement map and ledger layout. To view it:

1. Go to [map.blnkfinance.com](https://map.blnkfinance.com).
2. Click **Menu → Import**.
3. Paste the maps-tool JSON and load it.

If something does not match your product, say so in chat and wait for an updated map. When it looks right, tell the LLM to continue.
</Step>

<Step title="Ask it to implement">
When the map looks right, keep going in the same chat:

```text wrap
That map looks right. Implement it in this repo with the TypeScript SDK.
Use a Blnk Cloud sandbox for Core unless we are self-hosting.
```
</Step>

<Step title="Ask follow-ups as you go">
Stuck mid-build? Ask a normal follow-up in the same chat.

```text wrap
How should we hold a deposit until the bank confirms?
```
</Step>
</Steps>
