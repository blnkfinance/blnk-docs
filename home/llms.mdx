## Installing Skills

Skills teach your LLM how to design and implement Blnk using recommended best practices. Install them first so the LLM follows Blnk guidance instead of improvising.

<Steps>
<Step title="Add Skills">
From your project (or any directory where you use the LLM), run:

```bash wrap
npx skills add https://docs.blnkfinance.com
```

The CLI detects tools such as Cursor and Claude Code, and installs the skills for you. Confirm when prompted if you want them on more than one tool.
</Step>

<Step title="Confirm they loaded">
Ask your LLM to use the `blnk` skill. That is the root entry point; it routes to the other Blnk skills for architecture, money movement, and implementation.
</Step>
</Steps>

## Installing the Docs MCP

Connect the Docs MCP so your LLM can search and read these Docs while it builds with Skills.

The Docs MCP is available at:

```
https://docs.blnkfinance.com/mcp
```

<Steps>
<Step title="Add the Docs MCP">
Connect the Docs MCP in your LLM tool:

<Tabs>
<Tab title="Cursor">
1. Open the command palette with `Command + Shift + P` (or `Ctrl + Shift + P` on Windows).
2. Search for `Open MCP settings`.
3. Click **Add custom MCP**. This opens your `mcp.json` file.
4. Add the Docs MCP server:

```json wrap
{
  "mcpServers": {
    "blnk-docs": {
      "url": "https://docs.blnkfinance.com/mcp"
    }
  }
}
```

5. Save the file, then ask Cursor what MCP tools it has available. You should see the Docs MCP tools listed.
</Tab>

<Tab title="Claude Code">
Run:

```bash wrap
claude mcp add --transport http blnk-docs https://docs.blnkfinance.com/mcp
```

Confirm the server is connected:

```bash wrap
claude mcp list
```
</Tab>

<Tab title="VS Code">
1. Create a `.vscode/mcp.json` file in your project (or edit an existing one).
2. Add the Docs MCP server:

```json wrap
{
  "servers": {
    "blnk-docs": {
      "type": "http",
      "url": "https://docs.blnkfinance.com/mcp"
    }
  }
}
```
</Tab>

<Tab title="Claude">
1. Open **Settings → Connectors**.
2. Click **Add custom connector**.
3. Set the name to `Docs MCP` and the URL to `https://docs.blnkfinance.com/mcp`.
4. Click **Add**.
5. In a chat, use the attachments control and select the Docs MCP connector when you want Docs-backed answers.
</Tab>
</Tabs>

<Tip>
You can also install with `npx add-mcp https://docs.blnkfinance.com/mcp` if your LLM tool supports that flow.
</Tip>
</Step>

<Step title="Direct your LLM to use it">
Add this instruction to your project rules so the LLM reaches for the Docs MCP on every Blnk task:

```text Project instructions wrap
When working with Blnk, use the Docs MCP to search and read the official Docs before suggesting architecture, API calls, or code. Prefer Docs over prior knowledge.
```
</Step>
</Steps>

## Try it: build a wallet product

Once Skills and the Docs MCP are installed, you do not need a special ritual. Open a chat in your project and describe what you are building in plain language, the same way you would brief a teammate.

Say you are adding customer wallets: people deposit from a bank, hold a USD balance, and can send money to each other.

<Steps>
<Step title="Describe the product">
Paste something like this:

```text wrap
We're adding customer USD wallets.

- Each customer gets a wallet balance
- They deposit from a bank account
- They can transfer money to another customer
- No overdraft

Use the Blnk skills. Design the money movement first, then help me implement it. Look things up in the Docs MCP instead of guessing.
```

You do not need to name ledgers, balances, or API fields yourself. The Skills steer the LLM to map the money first; the Docs MCP keeps answers tied to the live guides.
</Step>

<Step title="Answer a few plain questions">
The LLM should ask short, product-level questions (for example who owns the funds before a deposit clears, or whether transfers are instant). Answer in normal language. Confirm defaults when they sound right.
</Step>

<Step title="Review the money map">
It should propose a money movement map and ledger layout before writing integration code. When it gives you maps-tool JSON, import it at [map.blnkfinance.com](https://map.blnkfinance.com) so you can see the flow. Correct anything that does not match your product, then tell it to continue.
</Step>

<Step title="Ask it to implement">
When the map looks right, keep going in the same chat:

```text wrap
That map looks right. Implement it in this repo with the TypeScript SDK against local Core, add happy-path tests, and save the decisions in .blnk_context/.
```

As it writes code, it should use the Docs MCP for endpoints, fields, and patterns instead of inventing them. You review the PR-style output the same way you would review a teammate's work.
</Step>
</Steps>

<Tip>
Stuck mid-build? Ask a normal follow-up: "How should we hold a deposit until the bank confirms?" or "Show me the create-transaction call for the P2P transfer." Skills and the Docs MCP stay in play for those questions too.
</Tip>
