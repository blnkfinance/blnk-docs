Blnk can be deployed in the way that best fits your infrastructure, operating model, and production requirements. Teams can start quickly with managed Cloud, run Blnk themselves, or work with Blnk on a private Cloud deployment with a production license.

## Deployment options

| Option | Best for | Operations model |
| --- | --- | --- |
| Managed Cloud | Teams that want the fastest path to a hosted Blnk instance | Blnk manages provisioning, hosting, upgrades, and core infrastructure operations |
| Private Cloud | Production teams that need a dedicated deployment, stronger isolation, or custom operating requirements | Blnk works with your team under a production license |
| Self hosting | Teams that need to run Blnk inside their own infrastructure | Your team provisions, secures, scales, monitors, and upgrades the environment |

## Managed Cloud

Managed Cloud is the simplest way to run Blnk without managing infrastructure yourself. Use it when you want Blnk to handle hosting, environment setup, routine operations, and platform updates so your team can focus on building financial products on top of the ledger.

## Private Cloud

Private Cloud is for teams that need a dedicated production deployment with stronger isolation, custom infrastructure requirements, or a closer operational partnership with Blnk. This option is built around a production license and fits companies with strict deployment, compliance, data residency, security, or procurement requirements.

## Self hosting

Self hosting lets you run Blnk in infrastructure your team controls. You choose where Blnk runs, how it is networked, how it is secured, and how it is scaled. Your team is responsible for provisioning infrastructure, configuring dependencies, managing uptime, monitoring performance, applying upgrades, and operating backups and recovery.

## Choosing an option

| Requirement | Recommended option |
| --- | --- |
| Fastest path to a hosted environment | Managed Cloud |
| Lowest infrastructure overhead | Managed Cloud |
| Dedicated production deployment | Private Cloud |
| Production license and Blnk-supported deployment planning | Private Cloud |
| Strict isolation, compliance, or data residency requirements | Private Cloud |
| Full control over infrastructure and networking | Self hosting |
| Running close to existing internal systems | Self hosting |
| Team-owned operations, monitoring, and upgrades | Self hosting |

Whatever deployment path you choose, teams can reach out to Blnk for pro support with architecture, production readiness, migration planning, and operational guidance.
