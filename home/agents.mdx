## Installing the Docs MCP

Connect the server once in your agent, then tell it to use Blnk Docs when you work on ledger features.

<Steps>
<Step title="Add the Docs MCP">
Connect the Blnk Docs MCP in your coding agent:

<Tabs>
<Tab title="Cursor">
1. Open the command palette with `Command + Shift + P` (or `Ctrl + Shift + P` on Windows).
2. Search for `Open MCP settings`.
3. Click **Add custom MCP**. This opens your `mcp.json` file.
4. Add the Blnk Docs server:

```json wrap
{
  "mcpServers": {
    "blnk-docs": {
      "url": "https://docs.blnkfinance.com/mcp"
    }
  }
}
```

5. Save the file, then ask Cursor what MCP tools it has available. You should see the Blnk Docs tools listed.
</Tab>

<Tab title="Claude Code">
Run:

```bash wrap
claude mcp add --transport http blnk-docs https://docs.blnkfinance.com/mcp
```

Confirm the server is connected:

```bash wrap
claude mcp list
```
</Tab>

<Tab title="VS Code">
1. Create a `.vscode/mcp.json` file in your project (or edit an existing one).
2. Add the Blnk Docs server:

```json wrap
{
  "servers": {
    "blnk-docs": {
      "type": "http",
      "url": "https://docs.blnkfinance.com/mcp"
    }
  }
}
```
</Tab>

<Tab title="Claude">
1. Open **Settings → Connectors**.
2. Click **Add custom connector**.
3. Set the name to `Blnk Docs` and the URL to `https://docs.blnkfinance.com/mcp`.
4. Click **Add**.
5. In a chat, use the attachments control and select the Blnk Docs connector when you want Docs-backed answers.
</Tab>
</Tabs>

<Tip>
You can also install with `npx add-mcp https://docs.blnkfinance.com/mcp` if your agent supports that flow.
</Tip>
</Step>

<Step title="Direct your agent to use it">
Add this instruction to your project agent rules so the agent reaches for Blnk Docs on every Blnk task:

```text Project instructions wrap
When working with Blnk, use the Blnk Docs MCP to search and read the official Docs before suggesting architecture, API calls, or code. Prefer Docs over prior knowledge.
```
</Step>
</Steps>

## What the MCP helps with

With the Docs MCP connected, your agent can:

- Search guides, tutorials, and API reference for a topic
- Read full pages when it needs exact request shapes or lifecycle details
- Cite the Docs it used so you can verify the guidance

Use it when you are designing ledgers, posting transactions, wiring SDKs, or checking configuration options. Keep your Core instance, API keys, and app code in your own environment; the Docs MCP only provides documentation context.
