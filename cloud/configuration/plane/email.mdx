Configure email transport so Plane can send login alerts, password resets, and other notifications.

---

## Email provider selection

```bash enterprise.env
EMAIL_PROVIDER=smtp
RESEND_API_KEY=
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `EMAIL_PROVIDER` | Email transport to use. Accepted values: `smtp` and `resend`. When unset, Plane uses SMTP if `SMTP_HOST` is set, otherwise Resend if `RESEND_API_KEY` is set. | Unset (auto-select) | No |
| `RESEND_API_KEY` | Resend API key. Required when `EMAIL_PROVIDER=resend`; also used when `EMAIL_PROVIDER` is unset and SMTP is not configured. | None | Conditional |

---

## SMTP transport

```bash enterprise.env
SMTP_HOST=smtp.resend.com
SMTP_PORT=587
SMTP_AUTH_USERNAME=resend
SMTP_AUTH_PASSWORD=re_your_api_key
SMTP_TLS_MODE=auto
SMTP_SKIP_TLS_VERIFY=false
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `SMTP_HOST` | SMTP server hostname from your provider. | None | Conditional |
| `SMTP_PORT` | SMTP port. Use `587` for STARTTLS or `465` for implicit TLS. | `587` | No |
| `SMTP_AUTH_USERNAME` | SMTP username. Required if `SMTP_AUTH_PASSWORD` is set. | None | Conditional |
| `SMTP_AUTH_PASSWORD` | SMTP password or API key from your provider. | None | No |
| `SMTP_TLS_MODE` | TLS mode for the connection. Accepted values: `auto` and `implicit`. | `auto` | No |
| `SMTP_SKIP_TLS_VERIFY` | Disable TLS certificate verification. Use `false` in production. | `false` | No |

<Tip>
With Resend, use host `smtp.resend.com`, port `587`, username `resend`, and your API key as the password. See [Resend SMTP documentation](https://resend.com/docs/send-with-smtp).
</Tip>
