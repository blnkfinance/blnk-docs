Tune how Plane proxies requests to Blnk Core instances.

---

## Timeout and retry settings

```bash enterprise.env
PROXY_TIMEOUT=30
PROXY_MAX_RETRIES=3
PROXY_RETRY_WAIT_MIN=1
PROXY_RETRY_WAIT_MAX=5
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `PROXY_TIMEOUT` | Request timeout in seconds. | `30` | No |
| `PROXY_MAX_RETRIES` | Maximum retry attempts for failed proxy requests. | `3` | No |
| `PROXY_RETRY_WAIT_MIN` | Minimum wait between retries in seconds. | `1` | No |
| `PROXY_RETRY_WAIT_MAX` | Maximum wait between retries in seconds. | `5` | No |

---

## Circuit breaker

```bash enterprise.env
PROXY_CIRCUIT_BREAKER_MAX_REQUESTS=100
PROXY_CIRCUIT_BREAKER_INTERVAL=60
PROXY_CIRCUIT_BREAKER_TIMEOUT=60
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `PROXY_CIRCUIT_BREAKER_MAX_REQUESTS` | Maximum requests allowed through a half-open circuit for each instance limiter. | `100` | No |
| `PROXY_CIRCUIT_BREAKER_INTERVAL` | Rolling interval in seconds used by each instance circuit breaker. | `60` | No |
| `PROXY_CIRCUIT_BREAKER_TIMEOUT` | Time in seconds a circuit remains open before trying half-open requests. | `60` | No |

---

## Rate limiting

```bash enterprise.env
PROXY_RATE_LIMIT_RPS=100
PROXY_RATE_LIMIT_BURST=200
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `PROXY_RATE_LIMIT_RPS` | Sustained request rate limit per connected instance. | `100` | No |
| `PROXY_RATE_LIMIT_BURST` | Burst capacity per connected instance. | `200` | No |
