Configure Plane authentication, encryption, and browser access controls.

---

## Runtime environment

```bash enterprise.env
ENVIRONMENT=production
AUTH_PROVIDER=local
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `ENVIRONMENT` | Runtime mode for Plane. Set to `production` for Production License deployments. Plane enables Sentry only when this is `production`. | None | No |
| `AUTH_PROVIDER` | Authentication provider. Accepted values: `local` and `workos`. When unset, Plane selects WorkOS if `WORKOS_API_KEY` and `WORKOS_CLIENT_ID` are set; otherwise it selects local auth when `JWT_SECRET` is set. | Auto-detect | No |

## Authentication and encryption

```bash enterprise.env
CORS_ORIGINS=https://ledger.yourdomain.com
BLNK_ENCRYPTION_KEY=0123456789abcdef0123456789abcdef
JWT_SECRET=your_long_random_secret_here
JWT_EXPIRY=24h
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `CORS_ORIGINS` | Comma-separated browser origins allowed to call the Plane API. No spaces between entries. | None | Conditional |
| `BLNK_ENCRYPTION_KEY` | 32-character key that encrypts sensitive data at rest, including instance connection keys. | None | Yes |
| `JWT_SECRET` | Secret used to sign local-auth session tokens. Required when `AUTH_PROVIDER=local` or when auth provider auto-detection falls back to local auth. | None | Conditional |
| `JWT_EXPIRY` | Local-auth session token lifetime. Go duration format. | `24h` | No |

### `CORS_ORIGINS`

Include `ENTERPRISE_PUBLIC_URL` and any other origins that load the Cloud Dashboard when the browser calls Plane cross-origin. Missing origins cause browser API requests to fail with CORS errors.

### `BLNK_ENCRYPTION_KEY`

Must be exactly 32 characters. Generate a unique value before your first deploy.

<Warning>
Do not change `BLNK_ENCRYPTION_KEY` after deployment. Rotating it prevents decryption of data encrypted with the previous key.
</Warning>

### `JWT_SECRET`

Use a long random string. Store it in your env file or secret manager, not in version control.

## Password reset

```bash enterprise.env
LOCAL_PASSWORD_RESET_URL=https://ledger.yourdomain.com/auth/reset-password
LOCAL_PASSWORD_RESET_EXPIRY=1h
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `LOCAL_PASSWORD_RESET_URL` | URL Plane embeds in password reset emails. When unset, Plane uses `{ENTERPRISE_PUBLIC_URL}/auth/reset-password`. | `{ENTERPRISE_PUBLIC_URL}/auth/reset-password` | No |
| `LOCAL_PASSWORD_RESET_EXPIRY` | Local-auth password reset token lifetime. Go duration format. | `1h` | No |

Set this explicitly when your reset page uses a different path. `ENTERPRISE_PUBLIC_URL` must be set if you leave `LOCAL_PASSWORD_RESET_URL` unset.

## WorkOS authentication

```bash enterprise.env
AUTH_PROVIDER=workos
WORKOS_API_KEY=
WORKOS_CLIENT_ID=
WORKOS_REDIRECT_URI=https://ledger.yourdomain.com/auth/finish-sso
WORKOS_PASSWORD_RESET_URL=https://ledger.yourdomain.com/auth/reset-password
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `WORKOS_API_KEY` | WorkOS API key. | None | Conditional |
| `WORKOS_CLIENT_ID` | WorkOS client ID. | None | Conditional |
| `WORKOS_REDIRECT_URI` | Redirect URI for WorkOS authentication flows. | None | Conditional |
| `WORKOS_PASSWORD_RESET_URL` | Password reset URL passed to WorkOS. | None | Conditional |

Set all four WorkOS variables when `AUTH_PROVIDER=workos`. If `AUTH_PROVIDER` is unset, setting `WORKOS_API_KEY` and `WORKOS_CLIENT_ID` makes Plane try WorkOS, and validation still requires `WORKOS_REDIRECT_URI` and `WORKOS_PASSWORD_RESET_URL`.
