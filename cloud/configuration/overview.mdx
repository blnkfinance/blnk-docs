Configure your Production License deployment with `enterprise.env`. The launcher reads this file, derives service-specific environment variables, and starts Core, Plane, Watch, the query agent, and the Cloud Dashboard UI.

---

## Getting started

To deploy for the first time, follow [Deploy the Production License](/cloud/start/deploy).

---

## Inherited configuration

Production License bundles multiple services. Some variables belong to the underlying services and are passed through by the launcher:

- **Core (`BLNK_*`)** — The launcher derives `BLNK_DATA_SOURCE_DNS` and `BLNK_REDIS_DNS` from your enterprise Postgres and Redis URLs. Uncomment `BLNK_*` vars only when overriding launcher defaults. See [Core configuration](/advanced/configuration/overview) and [Core reference](/advanced/configuration/reference).
- **Plane (`DB_*`, auth, email, proxy, deployment vars)** — The launcher derives Plane database settings from `ENTERPRISE_POSTGRES_URL`, then passes through direct Plane env vars such as auth, email, proxy, monitoring, and Kubernetes settings.
- **Watch (`WATCH_*`, `ALERT_*`)** — The launcher starts Watch and can auto-fill alert webhook credentials when Plane has an organization owner. See [Watch configuration](/watch/configuration).
- **Query agent (`DB_URL`, `CONNECTION_KEY`, `CONNECTION_URL`)** — The launcher derives these during managed startup. Set them directly only when running the query agent outside the launcher.
