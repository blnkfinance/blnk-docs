Configure the query agent that powers Insights and analytics queries in the Cloud Dashboard. In Production License launcher mode, most connection settings are derived automatically.

---

## Auto-provisioned connection

The launcher fills `DB_URL`, `CONNECTION_KEY`, `CONNECTION_URL`, and `HTTP_PORT` when it starts the query agent after Plane writes the bootstrap file. Set these directly only when running the query agent outside the launcher.

```bash enterprise.env
DB_URL=
CONNECTION_KEY=
CONNECTION_URL=
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `DB_URL` | PostgreSQL connection string for the query agent. In launcher mode, this is derived from the Core database URL. | Auto-filled by launcher | Required when direct |
| `CONNECTION_KEY` | Agent connection key. In launcher mode, this comes from Plane's query-agent bootstrap file. | Auto-filled by launcher | Required when direct |
| `CONNECTION_URL` | WebSocket URL the query agent uses to connect to Plane. | `wss://api.cloud.blnkfinance.com/ws/agent` when direct; launcher uses `ENTERPRISE_QUERY_AGENT_CONNECTION_URL` | No |

Control startup with `ENTERPRISE_SKIP_QUERY_AGENT`, `ENTERPRISE_SINGLE_QUERY_AGENT`, and `ENTERPRISE_QUERY_AGENT_*` on the [Launcher](/cloud/configuration/launcher) page.

## Query runtime

```bash enterprise.env
HTTP_PORT=9090
QUERY_TIMEOUT_MS=30000
DEBUG=false
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `HTTP_PORT` | Port the query agent listens on inside the container. | `9090` | No |
| `QUERY_TIMEOUT_MS` | Maximum query execution time in milliseconds. | `30000` | No |
| `DEBUG` | Enable debug logging for the query agent. | `false` | No |

---

## DuckDB and routing

```bash enterprise.env
ENABLE_DUCKDB=false
QUERY_BACKEND=
DUCKDB_PATH=
DUCKDB_DATA_DIR=
ENABLE_QUERY_ROUTING=false
```

| Variable | Description | Default | Required |
| :-- | :-- | :-- | :-- |
| `ENABLE_DUCKDB` | Enable DuckDB as a query backend. | `false` | No |
| `QUERY_BACKEND` | Query backend to use. Accepted values: `postgresql`, `duckdb`, and `auto`. Empty values default to PostgreSQL. | `postgresql` | No |
| `DUCKDB_PATH` | Parsed by configuration, but the current DuckDB backend creates `blnk_agent/blnk.db`. | None | No |
| `DUCKDB_DATA_DIR` | Parsed by configuration, but the current DuckDB backend uses `blnk_agent` and `blnk_agent/duckdb_temp`. | `blnk_agent` | No |
| `ENABLE_QUERY_ROUTING` | Enables query-router configuration. Current automatic query classification is limited; explicit request metadata can still select a backend. | `false` | No |
