---
title: "Update alert status"
api: "PUT /cloud/alerts/statuses/{key}"
description: "Update a custom alert status's name and/or colour."
---

import NeedHelp from "/cloud/snippets/need-help.mdx";

Update a custom alert status's name and/or colour. The status is identified by its key.

---

### Request structure

<RequestExample>
```bash cURL
curl -X PUT 'https://api.cloud.blnkfinance.com/alerts/statuses/PENDING_REVIEW' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Awaiting Review",
    "colour": "#10B981"
  }'
```
</RequestExample>

<ParamField path="key" type="string">
The status key to update (e.g. PENDING_REVIEW). Case-insensitive.
</ParamField>

<ParamField body="name" type="string" required>
New display name for the status.
</ParamField>

<ParamField body="colour" type="string" required>
New hex colour code. Must be one of the values in allowed_colours from the list statuses response.
</ParamField>

### Response structure

<ResponseExample>
```json Success
{
  "organization_id": "org_01ABC123",
  "key": "PENDING_REVIEW",
  "name": "Awaiting Review",
  "colour": "#10B981",
  "created_at": "2025-01-29T14:00:00Z",
  "updated_at": "2025-01-29T14:30:00Z"
}
```
</ResponseExample>

<ResponseField name="organization_id" type="string" required>
Organization ID that owns the status.
</ResponseField>

<ResponseField name="key" type="string" required>
The status key.
</ResponseField>

<ResponseField name="name" type="string" required>
Display name of the status (after update).
</ResponseField>

<ResponseField name="colour" type="string" required>
Hex colour code of the status (after update).
</ResponseField>

<ResponseField name="created_at" type="timestamp" required>
ISO 8601 timestamp when the status was created.
</ResponseField>

<ResponseField name="updated_at" type="timestamp" required>
ISO 8601 timestamp when the status was last updated.
</ResponseField>
