## Inflight transactions

A transaction will stay inflight indefinitely until it is acted upon or expired. There are two ways to act on an inflight transaction:

<img src="/cloud/img/transactions/inflight-actions.png" alt="Transaction action buttons including refund and actions menu" className="rounded-lg" />

- **Commit:** The transaction completes successfully, an `APPLIED` transaction is created and the funds move between balances.

- **Void:** The transaction is canceled, a `VOID` transaction is created and any holds are released.

### Commit transaction

<img src="/cloud/img/transactions/commit-transaction-modal.png" alt="Commit transaction modal with amount field" className="rounded-lg" />

To commit the full amount:

1. Select **Commit transaction** or press **C** on your keyboard. 
2. Leave Amount to commit empty.
3. Confirm in the modal.
4. A new linked transaction is created with its status updated to `APPLIED`.

For partial commits:
1. Enter the amount in **Amount to commit**.
2. A new linked transaction is created for that portion.
3. The rest remains inflight.
4. For the final commit, leave **Amount to commit** empty.

### Void transaction

Voiding permanently cancels an inflight transaction and releases any remaining balance. This cannot be undone. Both the transaction and any remaining inflight amounts are affected.

<img src="/cloud/img/transactions/void-transaction.png" alt="Void transaction confirmation modal" className="rounded-lg" />

1. Select **Void transaction** or press **V** on your keyboard.
2. Confirm in the modal.
3. A new linked transaction is created with its status updated to `VOID`.

---

## Refund transaction

Refunding sends funds back to the source balance.

<img src="/cloud/img/transactions/refund-transaction-modal.png" alt="Refund transaction confirmation modal" className="rounded-lg" />

1. Select **Refund transaction** or press **R** on your keyboard when on the transaction details page.
2. Confirm your action in the modal.
3. A new record is created with the refunded amount and status `APPLIED`.
