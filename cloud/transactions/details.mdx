The transaction details page provides a comprehensive view of individual transactions, displaying all relevant information from basic transaction data to inflight history and connected transactions.

---

## Accessing transaction details

Navigate to a transaction's details page by:

<img src="/cloud/img/transactions/transaction-details-page.png" alt="Main transaction details page layout" className="rounded-lg" />

1. Going to the `Transactions` section in your sidebar
2. Clicking on any transaction from the transactions list
3. The details page opens showing complete transaction information

---

## Transaction information

The main section displays essential transaction information in an organized format:

<img src="/cloud/img/transactions/transaction-information-section.png" alt="Transaction information section with all fields" className="rounded-lg" />

| Field | Description |
| :-- | :-- |
| **Amount** | Transaction amount prominently displayed at the top |
| **Currency** | Shows the transaction's denomination (USD, EUR, etc.) |
| **Status** | Current transaction state (`APPLIED`, `INFLIGHT`, `PENDING`, etc.) |
| **Source** | Origin balance or account for the transaction |
| **Destination** | Target balance or account receiving the funds |
| **Description** | User-defined description explaining the transaction purpose |
| `Date Created` | Timestamp when the transaction was initiated |
| `Effective Date` | When the transaction actually happened. If not specified, it will be the same as the date created |
| **Transaction ID** | Unique identifier for the specific transaction |
| **Reference** | Unique reference number for your transaction |
| **Parent Transaction** | Shows the parent transaction ID for related transactions |

## Transaction actions

The details page provides action buttons for transaction management:

- `Refund transaction`: Initiate a refund for the current transaction
- `Actions` menu: Access additional transaction operations

---

## Inflight history

<Note>This section is only visible for inflight transactions.</Note>

The inflight history section shows a list of all commits and voids for a particular inflight transaction.

<img src="/cloud/img/transactions/inflight-history-section.png" alt="Inflight history section for inflight transactions" className="rounded-lg" />

To "update" an inflight transaction, select `Commit` to commit it or `Void` to void it.

<Card title="Inflight transactions" iconType="duotone" icon="plane" href="/transactions/inflight">Learn more about managing inflight transactions.</Card>

## Metadata

The metadata section allows you to add custom information to your transaction. You can store additional details, tags, or any other data that helps you organize and manage your transactions.

### Edit metadata

To edit metadata:

<img src="/cloud/img/metadata/edit-metadata-button.png" alt="Edit metadata button in the Additional information section" className="rounded-lg" />

1. Navigate to the `Additional information` section.
2. Click the `Add` or `Edit` button at the top right of the section.
3. Use the code editor to add new metadata or update existing metadata.
4. Click `Update` to apply your changes.

<img src="/cloud/img/metadata/metadata-editor.png" alt="Metadata code editor interface for adding and editing custom data" className="rounded-lg" />

<Note>When updating metadata, any fields you don't include in your update will remain unchanged. Blnk adds your new values to the existing metadata instead of deleting them. To remove a value, set it to an empty string.</Note>
